# X-ON-X Knowledge Base

Юридична база ноу-хау на Google Sheets + Drive. Для **людей і AI-агентів** —
і як читачів, і як наповнювачів. Гібрид: Sheets = бібліотека (індекс + текстовий
**Опис + ключові слова**, що його читають пошук і агенти), Drive = самі файли.

## Структура — 3 полиці / 4 типи карток

| Полиця | Тип | Префікс ID | Тримає |
|--------|-----|-----------|--------|
| 1 Документи | **Прецеденти** | `PRE-` | реальні кейси/угоди: анонімізована копія + звʼязки |
| 1 Документи | **Шаблони** | `TPL-` | багаторазові документи з плейсхолдерами |
| 2 Рісьорчі | **Рісьорчі** | `RES-` | меморандуми/інсайти: питання + висновок |
| 3 Провайдери | **Провайдери** | `PRV-` | зовнішні юристи/вендори: тип, контакти, оцінка |

Поля — українські, узгоджені з дизайном (Notion). Категорія = 1.1–1.6 (тип
відносин), Юрисдикція = UA/DE/US-DE/UK/EU/Multi, Оцінка = Recommended/Okay/Avoid.

**Службові поля для AI** (на кожній картці): `Опис + ключові слова` (обовʼязкове,
це читає пошук і AI), `Статус` (active / pending-PII / needs-review — AI бачить
лише `active`), `Embedding` (схована колонка, вектор для семантики — поки порожня),
`Звʼязки` (двосторонні peer-лінки), Хто створив / Створено / Оновлено.

Наповнення йде через вкладку **Inbox** (чернетки → ревʼю → promote). `_Taxonomy`
тримає дропдауни і валідацію. `Home` документує все в самому шиті.

## Файли

| Файл | Роль |
|------|------|
| `kb_client.py` | спільна авторизація (gspread + Drive v3) на кешованому OAuth-токені |
| `kb_schema.py` | єдине джерело правди: таблиці, колонки, таксономія, правила ID |
| `build_kb.py` | one-shot білдер: Drive-дерево + спредшит + стилі + seed |
| `kb_api.py` | рантайм-контракт для людей і агентів (search/get/propose/promote/link/update/clear_pii/embed_all) |
| `kb_search.py` | пошуковий шар: TF-IDF ранжування (без ключа) + гачок під embeddings (Voyage/OpenAI) |
| `kb_sortuvalnyk.py` | **Сортувальник**: розбирає кинуте → класифікує → propose() в Inbox |
| `kb_demo.py` | наскрізне демо агента |
| `kb_config.json` | пише білдер: spreadsheet id + Drive folder ids |

## API (`from kb_api import KB`)

```python
kb = KB()
kb.search("warranty cap", table=None, status="active")   # знайти (за замовч. лише active)
kb.get("PRE-0001")                                        # картка + повʼязані
kb.propose("Шаблони", {...}, proposed_by="agent:x")       # чернетка → Inbox
kb.promote("INB-0001", reviewer="Iryna")                  # Inbox → таблиця, реальний ID
kb.reject("INB-0002", reviewer="Iryna", notes="дубль")
kb.link("PRE-0001", "PRV-0002")                           # двосторонній звʼязок
kb.update("RES-0001", {"Статус": "archived"})
kb.clear_pii("PRE-0003")                                  # pending-PII → active
kb.embed_all()                                            # порахувати вектори (потрібен провайдер)
```

**Пошук** ранжує за релевантністю (`search` повертає `_score` + `_match`):
lexical TF-IDF без ключа; векторний (за змістом, крос-мовний) — щойде колонка
`Embedding` заповнена і є `VOYAGE_API_KEY`/`OPENAI_API_KEY` (тоді `embed_all()`
рахує вектори).

**PII-флоу:** `promote()` кладе Прецеденти як `pending-PII` (AI їх не бачить),
поки `clear_pii()` або `promote(..., pii_ok=True)` не підтвердить анонімізацію.
Решта типів — одразу `active`.

Контракт тримається в `kb_api` (не в шиті): `Опис + ключові слова` обовʼязковий,
дропдаун-поля валідуються по `_Taxonomy`, ID генеруються, звʼязки пишуться з обох
боків, `Оновлено` штампується, на promote прикріплений Drive-файл переїжджає у
відповідну секційну папку.

## Сортувальник (`from kb_sortuvalnyk import Sortuvalnyk`)

```python
s = Sortuvalnyk()
s.intake("Олена Коваль, перекладач, Львів, +380...")   # → INB-XXXX (тип/категорія/опис авто)
```

Два режими: **LLM** (Anthropic Claude, якщо є `ANTHROPIC_API_KEY`) і
**fallback за маркерами** (детермінований, працює без ключа). Визначає тип,
категорію, юрисдикцію, пише опис, оцінює впевненість і кладе чернетку в Inbox.

## Перебудова

`python3 build_kb.py` створює **новий** спредшит + дерево папок щоразу (one-shot).
Дані наповнюй через `kb_api` / `kb_sortuvalnyk`, не перезапуском білда.

Авторизація: той самий кешований OAuth-токен, що й решта sheets-тулінгу
(`~/.claude-sheets/token.json`).
