# X-ON-X Knowledge Base — фронт «бібліотека» на AppSheet

Мета: на той самий KB-лист накласти AppSheet — щоб переглядати/шукати як у
бібліотеці (картки, секції, фільтри), без переносу даних. Лист лишається
джерелом правди + редактором; AppSheet — гарний фронт «тільки перегляд/пошук».

Створення додатку відбувається у **твоєму** AppSheet-акаунті (Google) —
зробити це за тебе ззовні не можна. Кроки нижче — ~10–15 хв.

Бренд: графіт `#111116`, лайм-акцент `#5A7A00`, кремовий фон `#FAFAF5`.

## 1. Створити додаток
- Відкрий Google Sheet → меню **Extensions → AppSheet → Create an app**
  (або зайди на appsheet.com → Create → App → With existing data → обери цей файл).
- AppSheet просканує вкладки. Залиш як таблиці лише полиці:
  **Прецеденти, Шаблони, Рісьорчі, Провайдери.**
  Вкладки `Home`, `_Index`, `_Taxonomy`, `Inbox` — **видали з таблиць** (Data →
  прибрати), вони службові. (Inbox можеш лишити окремою таблицею, якщо хочеш
  бачити чергу на рев'ю в додатку.)

## 2. Типи колонок (Data → Columns, для кожної таблиці)
- `Файл`, `Папка на Drive` → тип **Url** (стають клікабельними).
- `Опис` → **LongText**; `Ключові слова` → **Text**.
- `Перевірений локалом`, `Підтверджено локалом`, `Партнер` → **Yes/No**.
- `Категорія / Тип документа / Право / Сфера / Форма / Тип послуги / Оцінка /
  Юрисдикція / регіон` → **Enum** (AppSheet підхопить значення; за бажання
  встав списки з вкладки `_Taxonomy`).
- Сховай службові: зніми галочку **Show?** для `ID, Статус, Embedding, Звʼязки,
  Хто створив, Створено, Оновлено`.

## 3. Вигляд «бібліотека» (UX → Views) — по одному на полицю
Для кожної полиці зроби **Deck** view (або Gallery):
- **Прецеденти / Шаблони:** Primary header = `Назва`; Secondary = `Категорія`;
  Summary = `Опис`; додай у деталі `Тип документа, Право, Файл`.
- **Рісьорчі:** header = `Назва`; secondary = `Сфера`; summary = `Опис`;
  деталі `Форма, Право, Питання / тригер, Файл`.
- **Провайдери:** header = `Назва`; secondary = `Тип послуги`; summary = `Опис`;
  деталі `Юрисдикція / регіон, Оцінка, Контакти, Папка на Drive`.
- Кожен view: **Position = center**, іконку секції на смак (book / doc / people).

## 4. Тільки актуальне (сховати pending-PII з загального перегляду)
- Data → **Slices** → New slice `Активні` на кожній полиці, фільтр:
  `[Статус] = "active"`. У Deck-view постав Source = цей slice.
  (Прецеденти з PII не світяться, поки не знято PII — як і для AI.)

## 5. Пошук і фільтри
- У кожному Deck-view увімкни **Search** (UX → view → Search on).
- **Quick edit / Filter:** додай фільтр-колонки `Категорія`, `Право`
  (для провайдерів — `Тип послуги`, `Юрисдикція / регіон`).
- Хочеш єдиний пошук по всьому — зроби view типу **Dashboard** з 4 секціями.

## 6. Бренд
- UX → **Format Rules / Theme**: Primary color `#5A7A00` (лайм) або `#111116`
  (графіт), світла тема, шрифт за смаком. Лого X-ON-X у Settings → App logo.

## 7. (Опц.) Іконка-обкладинка на картці
Зроби **virtual column** `Іконка` (Data → Columns → Add virtual column),
формула напр.:
```
IF([Статус]="pending-PII", "⏳", "✓")
```
або емодзі по полиці (📄 Прецеденти / 📋 Шаблони / 🔍 Рісьорчі / 👤 Провайдери).
Признач її як label у Deck-view.

## 8. Поділитися з командою
- Settings → **Users** / Share → додай команду (Workspace-логіни VK, OK, …).
- Деплой: **Manage → Deploy** (для >10 користувачів — Core plan, який уже
  входить у ваш Google Workspace).

---

### Що лишається в самому листі (не в AppSheet)
- **Наповнення** через Inbox (бот) і **«approved»** — це у Sheets.
- AppSheet = перегляд/пошук/фільтри. Можна додати й редагування карток
  у додатку, якщо захочеш (увімкнути Edit на view).
